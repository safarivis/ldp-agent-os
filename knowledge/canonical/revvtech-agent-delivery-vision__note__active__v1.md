# RevvTech Agent Delivery Vision

## KISS statement

We are building a practical delivery operating system for ourselves and our clients.

Each real project should have its own repo as the source of truth for the work.
A shared `_agent` layer should hold the reusable standards, workflows, templates, and learnings that help us run many projects better over time.
Agents like `pi` should work inside that structure to help with execution, continuity, and pattern capture.
Humans should remain in control of approvals, sensitive actions, and major decisions.

## Why this matters

Most value from AI agents will not come from isolated chats alone.
It will come from agents working inside real business operating structures with:
- clear context
- durable memory
- repeatable workflows
- tool access
- permissions and approvals
- reusable patterns

The goal is not to sound advanced.
The goal is to reduce chaos, improve handover, speed up setup, and compound useful learnings across projects.

## Simple model

### Project repo
Use the repo for project-specific truth:
- code
- setup
- tasks
- status
- decisions
- daily notes

### `_agent`
Use `_agent` for reusable operating truth:
- templates
- workflows
- preferred patterns
- cross-project learnings
- pattern candidates and promotions

### Human layer
Use email, messaging, and meetings for:
- approvals
- alignment
- commercial decisions
- sensitive external communication

Then write the outcome back into the repo or `_agent`.

## North-star outcomes

Success looks like:
- faster setup for each new project
- better continuity across humans and agents
- less re-solving of the same operational problems
- clearer project state at any point in time
- reusable standards that improve delivery across multiple clients

## KISS operating rule

If something is true only for one project, keep it in that project repo.
If it is useful across multiple projects, promote it into `_agent`.
If a human decision changes the direction, record it back into the system.

## Pattern loop

At the end of meaningful work, ask:
- "Anything here we should standardize or reuse?"
- or: "Any patterns?"

Capture likely candidates simply first.
Promote only the clear winners.

## Guardrail

This only creates real value if it improves actual delivery.
If it is not reducing friction, improving handover, or increasing reuse, then it is just theory and should be simplified.
