# Memory Templates

Use these templates when setting up or refreshing the global memory core for a root `_agent` workspace.

Files:
- `soul.md` — identity, philosophy, non-negotiables, and the Six Pillars
- `user.md` — user preferences, working style, and constraints
- `context.md` — current priorities, operating focus, open loops, and durable operational notes
- `telos.md` — long-term direction and explicit tradeoff preferences

KISS rule:
- keep memory concrete and useful
- prefer explicit active preferences over placeholders
- prefer current operating focus over generic backlog lists
- update live memory when repeated eval wins show a better default

See also:
- `ROOT-SETUP.md` — how to bootstrap a fresh `_agent` memory core from these templates
