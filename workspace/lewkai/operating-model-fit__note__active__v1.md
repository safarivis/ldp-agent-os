# Lewkai Operating Model Fit

## KISS summary

Lewkai fits the same operating model being shaped with RevvTech:
- each real client/project should have its own repo as local truth
- `_agent` should hold reusable standards, templates, workflows, and cross-project learnings
- agents should work against structured files and operating context, not only chat
- useful patterns discovered in Lewkai delivery should be promoted into reusable assets for future projects

## Why relevant to Lewkai
Lewkai is positioning around AI implementation, systems improvement, and practical delivery.
That means the delivery model itself is part of the value:
- clearer project setup
- better handover
- repeatable implementation patterns
- compounding learnings across clients

## Practical implication
Lewkai should not be framed only as advice or prompts.
It should increasingly be framed as a structured implementation practice with:
- project repos
- operating notes
- repeatable workflows
- agent-assisted execution
